# Artifact Service - Logic Contract

## Business Rules

### ART-BR-001: Artifact Creation

- `title` must contain non-whitespace text.
- `content_type` must be present.
- First version content is required.
- Created artifacts default to private visibility unless explicitly set.
- The current version points at the first persisted version.

### ART-BR-002: Ownership

- Owner-only operations require `artifact.owner_user_id == caller_user_id`.
- Non-owner callers cannot update, delete, publish, revoke, list shares, or add
  versions.
- Private artifacts cannot be read by non-owners through the library API.

### ART-BR-003: Versioning

- Versions are immutable after creation.
- New versions increment from the current maximum version number unless an
  explicit valid number is provided by repository code.
- Share version pins must reference an existing version number.

### ART-BR-004: Sharing

- Share tokens are generated by the service and must be treated as secrets.
- Revoked or expired shares are invalid.
- Organization shares require an org id from the request or artifact owner org.
- Organization share reads require the caller org to match the share org.

### ART-BR-005: Remix

- Remix requires a valid active share.
- The remixed artifact is private and owned by the caller.
- Remix preserves lineage through `parent_artifact_id`.
- Remix copies the selected shared version into the new artifact's first version.

### ART-BR-006: Runtime Quota

- Runtime invocation must load and validate the artifact first.
- Prompt text is required.
- Daily usage is counted per artifact/user/day.
- When daily calls reach quota, the service raises a quota error with
  `Retry-After` seconds.
- Model failures fall back to the documented stub output.

### ART-BR-007: MCP Approval

- MCP approval requires an existing artifact.
- MCP calls without an active `allow` + `always` grant return
  `requires_approval=true`.
- Granted MCP calls touch `last_used_at`.
- MCP transport failures fall back to a stub result.

### ART-BR-008: KV Scope

- `scope=personal` requires `user_id`.
- Personal reads and writes are isolated by user id.
- `scope=shared` writes require `artifact.storage_scope=shared`.
- Shared responses expose `user_id=null`.

## State Machines

### Artifact Visibility

```text
private -> unlisted
private -> org
private -> public
unlisted -> private
org -> private
public -> private
```

Visibility changes update the artifact metadata only; share-token validity is
controlled by share records.

### Share State

```text
active -> revoked
active -> expired
revoked -> terminal
expired -> terminal
```

## API Contracts

- Route-layer domain errors map to 400, 403, or 404.
- Unexpected errors log and return 500.
- Successful mutation routes return the updated domain model or action result.
- Runtime and MCP routes return stable response schemas even on upstream
  dependency failure.
